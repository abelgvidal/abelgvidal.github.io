The pod is marked NotReady and feels the same.

Posted: 0001-01-01   |   Tags: kubernetes

When K8s wants to end a pod, it gives it the change to end “gracefully”. The preStop hook (if exists) is executed before killing the pod. This is a great place to put: code that waits for connections to close notifies other components cleans up stuff k8s it marks the pod as status=TERMINATING pods is marked as NotReady Because is marked as NotReady its endpoint is erased from service endpoints (no more traffic received) Then, it sends to the pod a signal SIGTERM to the main process in each container of the pod.
